Notification API

실시간 알림 API - System Server

패키지: kr.fingate.ef.system.api.notification.controller
Base Path: /api/{version}/notifications

SSE vs Polling

알림 시스템은 두 가지 방식을 제공합니다:

  • SSE (Server-Sent Events): 실시간 양방향 연결, 권장 방식
  • Polling: 네트워크 제약 환경(방화벽, 프록시)을 위한 대체 방식

1. SSE 알림 구독 인증 필요 SSE

URL /{clientType}/{version}/notifications/subscribe
Method GET
Content-Type text/event-stream

API 설명

  • 목적: 실시간 알림 수신을 위한 SSE 연결 수립
  • 선행 조건: JWT 인증 필요
  • 기대 효과: 서버에서 발생하는 알림을 실시간으로 수신

Request Headers

헤더필수설명
Authorization필수Bearer {JWT 토큰}
Last-Event-ID선택재연결 시 마지막으로 수신한 이벤트 ID (유실 방지)

SSE 이벤트 형식

event: connected data: {"sessionId": "uuid", "userNo": 1234} event: notification id: evt-12345 data: {"eventId": "evt-12345", "eventType": "PASSWORD_CHANGED", ...} event: heartbeat data: {"timestamp": "2026-01-18T12:00:00Z"}

Last-Event-ID 재연결

네트워크 단절 후 재연결 시 Last-Event-ID 헤더를 포함하면, 서버가 해당 이벤트 이후의 누락된 알림을 자동으로 전송합니다.

2. 알림 목록 조회 (Polling) 인증 필요

URL /{clientType}/{version}/notifications
Method GET

API 설명

  • 목적: SSE 대신 주기적으로 새 알림 확인 (Polling 방식)
  • 선행 조건: JWT 인증 필요
  • 기대 효과: SSE 연결이 불가능한 환경에서도 알림 수신 가능

Query Parameters

파라미터타입필수설명
afterIdString선택이 ID 이후의 알림만 조회
limitInteger선택조회할 최대 개수 (기본값: 20)

성공 응답

{ "code": 1, "message": "OK", "data": { "items": [ { "eventId": "evt-12345", "eventType": "PASSWORD_CHANGED", "userNo": 1234, "channel": "SECURITY", "title": "비밀번호 변경", "message": "비밀번호가 변경되었습니다.", "payload": {...}, "occurredAt": "2026-01-18T12:00:00Z", "read": false } ], "lastEventId": "evt-12345", "count": 1, "hasMore": false } }

3. 알림 단건 조회 인증 필요

URL /{clientType}/{version}/notifications/{eventId}
Method GET

Path Parameters

파라미터타입필수설명
eventIdString필수알림 이벤트 ID

성공 응답

{ "code": 1, "message": "OK", "data": { "eventId": "evt-12345", "eventType": "PASSWORD_CHANGED", "userNo": 1234, "channel": "SECURITY", "title": "비밀번호 변경", "message": "비밀번호가 변경되었습니다.", "payload": {...}, "occurredAt": "2026-01-18T12:00:00Z", "read": false, "readAt": null } }

4. 알림 읽음 처리 인증 필요

URL /{clientType}/{version}/notifications/{eventId}/read
Method POST

Path Parameters

파라미터타입필수설명
eventIdString필수읽음 처리할 알림 이벤트 ID

성공 응답

{ "code": 1, "message": "OK", "data": null }

5. 전체 알림 읽음 처리 인증 필요

URL /{clientType}/{version}/notifications/read-all
Method POST

API 설명

  • 목적: 현재 사용자의 모든 알림을 읽음 처리
  • 선행 조건: JWT 인증 필요
  • 기대 효과: 알림 목록에서 모든 알림이 읽음 상태로 변경

성공 응답

{ "code": 1, "message": "OK", "data": null }
