CommonCodeController

공통코드 조회 API - System Server

패키지: kr.fingate.ef.system.api.commoncode.controller
Base Path: /api/{version}/common-codes

1. 그룹코드로 공통코드 목록 조회 공개

URL /{clientType}/{version}/common-codes/groups/{groupCode}
Method GET

API 설명

  • 목적: 특정 그룹코드에 속하는 공통코드 목록 조회
  • 선행 조건: 없음 (공개 API)
  • 기대 효과: 드롭다운, 선택 목록 등에 사용할 공통코드 데이터 획득

Path Parameters

파라미터타입필수설명
clientTypeString필수클라이언트 타입 (app, web)
versionString필수API 버전 (v1, v2, ...)
groupCodeString필수공통코드 그룹코드

성공 응답

{ "code": 1, "message": "OK", "data": { "groupCode": "GENDER", "groupName": "성별", "items": [ { "groupCode": "GENDER", "code": "M", "codeName": "남성", "sortOrder": 1, "useYn": "Y" }, { "groupCode": "GENDER", "code": "F", "codeName": "여성", "sortOrder": 2, "useYn": "Y" } ] } }

2. 그룹코드와 코드로 단건 조회 공개

URL /{clientType}/{version}/common-codes/groups/{groupCode}/codes/{code}
Method GET

API 설명

  • 목적: 특정 그룹코드와 코드에 해당하는 공통코드 단건 조회
  • 선행 조건: 없음 (공개 API)
  • 기대 효과: 코드명, 설명 등 상세 정보 획득

Path Parameters

파라미터타입필수설명
groupCodeString필수공통코드 그룹코드
codeString필수공통코드 코드

성공 응답

{ "code": 1, "message": "OK", "data": { "groupCode": "GENDER", "code": "M", "codeName": "남성", "sortOrder": 1, "useYn": "Y" } }

3. 여러 그룹코드 일괄 조회 공개

URL /{clientType}/{version}/common-codes/groups/batch
Method POST

API 설명

  • 목적: 여러 그룹코드를 한 번에 조회하여 네트워크 호출 최소화
  • 선행 조건: 없음 (공개 API)
  • 기대 효과: 페이지 로드 시 필요한 공통코드를 일괄 획득

Request Body

["GENDER", "STATUS", "REGION"]

성공 응답

{ "code": 1, "message": "OK", "data": [ { "groupCode": "GENDER", "groupName": "성별", "items": [...] }, { "groupCode": "STATUS", "groupName": "상태", "items": [...] } ] }

4. 전체 공통코드 조회 공개

URL /{clientType}/{version}/common-codes/all
Method GET

API 설명

  • 목적: 전체 공통코드를 그룹별로 정리하여 조회
  • 선행 조건: 없음 (공개 API)
  • 기대 효과: 클라이언트 측 공통코드 캐시 구축

성공 응답

{ "code": 1, "message": "OK", "data": { "GENDER": [...], "STATUS": [...], "REGION": [...] } }

5. 캐시 갱신 (관리자용) 인증 필요

URL /{clientType}/{version}/common-codes/cache/refresh
Method POST

API 설명

  • 목적: 공통코드 캐시를 강제로 갱신
  • 선행 조건: 관리자 권한 필요
  • 기대 효과: DB에서 변경된 공통코드가 즉시 반영

성공 응답

{ "code": 1, "message": "OK", "data": null }
