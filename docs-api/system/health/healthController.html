HealthController

헬스체크 API - System Server

패키지: kr.fingate.ef.system.api.health
Base Path: /health

Kubernetes 연동

이 API들은 Kubernetes의 Liveness/Readiness Probe에서 사용됩니다:

  • /health/live: livenessProbe - 컨테이너 재시작 여부 결정
  • /health/ready: readinessProbe - 트래픽 수신 가능 여부 결정

1. 서버 상태 확인 공개

URL /health
Method GET

API 설명

  • 목적: 서버의 기본 상태 및 정보 확인
  • 선행 조건: 없음 (공개 API, Gateway 인증 불필요)
  • 기대 효과: 서버 동작 여부와 기본 정보 확인

성공 응답

{ "code": 1, "message": "OK", "data": { "status": "UP", "timestamp": "2026-01-18T12:00:00", "service": "platform" } }

2. Readiness 체크 공개 K8s

URL /health/ready
Method GET

API 설명

  • 목적: 서버가 요청을 처리할 준비가 되었는지 확인
  • 선행 조건: 없음
  • Kubernetes: readinessProbe에서 사용
  • 기대 효과: READY가 아닌 경우 로드밸런서에서 트래픽 제외

성공 응답

{ "code": 1, "message": "OK", "data": { "status": "READY" } }

3. Liveness 체크 공개 K8s

URL /health/live
Method GET

API 설명

  • 목적: 서버가 살아있는지 확인
  • 선행 조건: 없음
  • Kubernetes: livenessProbe에서 사용
  • 기대 효과: ALIVE가 아닌 경우 컨테이너 재시작

성공 응답

{ "code": 1, "message": "OK", "data": { "status": "ALIVE" } }
