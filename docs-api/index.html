Gateway 접근 정보

모든 API는 Gateway를 통해 접근해야 합니다. URL 패턴은 다음과 같습니다:

/{clientType}/{version}/{endpoint}

파라미터 값 설명
{clientType} app, web 클라이언트 유형 식별자
{version} v1, v2, ... API 버전

Gateway 포트: 9999

참고: Gateway가 내부적으로 /app/v1/** → /api/v1/**로 URL을 변환합니다.

지원 API 버전

  • v1 - 현재 안정 버전
  • v2 - 확장 기능 지원

API 개요

  • 모든 API는 POST 메서드만 사용합니다
  • 요청 및 응답 본문은 JSON 형식입니다
  • 인증은 Gateway에서 JWT를 통해 처리됩니다
  • 공개 API (회원가입, 로그인, 인증)는 JWT 인증이 필요하지 않습니다
  • 로그인 시 비밀번호는 RSA 공개키로 암호화하여 전송해야 합니다

Gateway에서 발생할 수 있는 오류 케이스입니다. Application API 호출 전에 Gateway 단계에서 차단될 수 있습니다.
모든 Gateway 오류는 Application과 동일한 JSON 응답 구조로 반환됩니다.

통일된 응답 구조

성공/실패 여부와 관계없이 모든 API는 동일한 구조로 응답합니다.

// 성공 (code == 1) { "code": 1, "message": "OK", "data": { ... } } // 실패 (code != 1) { "code": 에러코드(int), "message": "에러 메시지", "data": null }

Frontend 처리: code == 1이면 성공, code != 1이면 message를 사용자에게 노출

JWT 인증 오류 (11XXX)

HTTP 401 Unauthorized ▼

1 Authorization 헤더 누락 (code: 11001)

요청에 Authorization 헤더가 없거나 비어있는 경우

{"code": 11001, "message": "인증 토큰이 없습니다.", "data": null}
해결: 로그인 API로 JWT 토큰을 발급받은 후, Authorization: Bearer {token} 헤더를 추가하세요.

2 Bearer 토큰 형식 오류 (code: 11002)

Authorization 헤더가 Bearer 로 시작하지 않거나 토큰 형식이 잘못된 경우

{"code": 11002, "message": "토큰 형식이 올바르지 않습니다.", "data": null}
해결: 헤더 값이 Bearer {토큰} 형식인지 확인하세요.

3 JWT 토큰 만료 (code: 11003)

토큰의 유효 기간이 지난 경우 (exp claim 초과)

{"code": 11003, "message": "토큰이 만료되었습니다.", "data": null}
해결: 로그인 API를 다시 호출하여 새 토큰을 발급받으세요.

4 JWT 서명 검증 실패 (code: 11004)

토큰이 위조되었거나 잘못된 secret key로 생성된 경우

{"code": 11004, "message": "토큰 서명이 유효하지 않습니다.", "data": null}
해결: 로그인 API에서 발급받은 정상 토큰을 사용하세요.

5 JWT Issuer 불일치 (code: 11005)

토큰의 발급자(iss) 클레임이 일치하지 않는 경우

{"code": 11005, "message": "토큰 발급자가 유효하지 않습니다.", "data": null}
해결: 다른 시스템에서 발급된 토큰은 사용할 수 없습니다.

6 JWT Audience 불일치 (code: 11006)

토큰의 대상(aud) 클레임이 일치하지 않는 경우

{"code": 11006, "message": "토큰 대상이 유효하지 않습니다.", "data": null}
해결: 올바른 대상으로 발급된 토큰을 사용하세요.

API 버전 정책 오류 (12XXX)

HTTP 403 / 405 / 410 ▼

1 클라이언트-API 조합 차단 (code: 12001)

특정 클라이언트 타입에서 해당 API 버전 사용이 금지된 경우 (HTTP 403)

{"code": 12001, "message": "해당 API 버전은 차단되었습니다.", "data": null}
해결: 해당 클라이언트 타입에서 지원하는 API 버전을 사용하세요.

2 읽기 전용 모드 (code: 12002)

해당 API 버전이 읽기 전용 모드로 전환된 경우 (HTTP 405)

{"code": 12002, "message": "해당 API 버전은 읽기 전용입니다.", "data": null}
해결: 새 버전의 API를 사용하거나, GET/HEAD/OPTIONS 요청만 가능합니다.

3 서비스 종료된 버전 (code: 12003)

해당 API 버전이 완전히 종료된 경우 (HTTP 410)

{"code": 12003, "message": "해당 API 버전의 서비스가 종료되었습니다.", "data": null}
해결: 현재 지원되는 API 버전으로 마이그레이션하세요.

4 지원하지 않는 버전 (code: 12004)

요청한 API 버전을 지원하지 않는 경우

{"code": 12004, "message": "지원하지 않는 API 버전입니다.", "data": null}
해결: 지원되는 API 버전(v1, v2)을 사용하세요.

! Deprecation 경고 (WARNING)

해당 API 버전이 곧 종료될 예정임을 알리는 경고 (요청은 정상 처리됨)

HTTP/1.1 200 OK | X-Gateway-Warning: API_VERSION_DEPRECATED
권장: 새 버전으로 마이그레이션을 준비하세요.

라우팅 오류 (13XXX)

HTTP 404 Not Found ▼

1 존재하지 않는 경로 (code: 13001)

등록되지 않은 API 엔드포인트로 요청한 경우

{"code": 13001, "message": "요청한 경로를 찾을 수 없습니다.", "data": null}
해결: API 문서에서 정확한 엔드포인트 URL을 확인하세요.

2 잘못된 클라이언트 타입 (code: 13002)

허용되지 않은 클라이언트 타입으로 요청한 경우 (예: /admin/v1/...)

{"code": 13002, "message": "허용되지 않은 클라이언트 타입입니다.", "data": null}
해결: 허용된 클라이언트 타입(app, web)을 사용하세요.

3 Gateway 우회 시도

시스템 경로를 통해 직접 접근을 시도한 경우

{"code": 13001, "message": "요청한 경로를 찾을 수 없습니다.", "data": null}
해결: /api/...가 아닌 /app/... 또는 /web/... 경로를 사용하세요.

서버 오류 (19XXX)

HTTP 500 / 503 ▼

1 Application 서버 연결 실패 (code: 19001)

백엔드 Application 서버에 연결할 수 없는 경우 (HTTP 503)

{"code": 19001, "message": "백엔드 서버에 연결할 수 없습니다.", "data": null}
원인: Application 서버가 다운되었거나 네트워크 문제가 발생했습니다.

2 System 서비스 일시 불가 (code: 19002)

시스템 서비스가 일시적으로 사용 불가능한 경우 (HTTP 503)

{"code": 19002, "message": "시스템 서비스가 일시적으로 사용 불가합니다.", "data": null}
원인: 시스템 점검 또는 일시적인 과부하 상태입니다.

3 Gateway 내부 오류 (code: 19999)

Gateway 처리 중 예기치 않은 오류가 발생한 경우 (HTTP 500)

{"code": 19999, "message": "Gateway 내부 오류가 발생했습니다.", "data": null}
원인: 서버 관리자에게 문의하세요.
