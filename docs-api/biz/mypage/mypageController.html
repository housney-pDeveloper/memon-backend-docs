MypageController API 문서

소스 경로: biz/mypage/controller/MypageController.java

인증 필수

이 Controller의 모든 API는 로그인 상태에서만 호출 가능합니다.

요청 시 Authorization: Bearer {accessToken} 헤더가 필수입니다.

참고: 휴대전화 변경 SMS 인증은 MobileAuthController에서 처리합니다.
  • 인증번호 발송: POST /api/{version}/mobileAuth/insAuthSmsAuth (authType=CHANGE_PHONE)
  • 인증번호 검증: POST /api/{version}/mobileAuth/updAuthSmsAuthVerify (authType=CHANGE_PHONE)

2. 휴대전화번호 변경 - 최종 변경 (updChangePhone)

휴대전화번호 변경 프로세스

  1. MobileAuthController: insAuthSmsAuth (authType=CHANGE_PHONE) - 인증번호 발송
  2. MobileAuthController: updAuthSmsAuthVerify (authType=CHANGE_PHONE) - 인증번호 검증
  3. [현재 API] 최종 변경 - authKey로 휴대전화번호 변경

3. 비밀번호 변경 (updChangePassword)

API URL

POST
/{clientType}/{version}/mypage/updChangePassword

인증 및 권한

토큰ACCESS_TOKEN
권한USER
HeaderAuthorization: Bearer {accessToken}

Request Body (ChangePasswordDto)

필드명타입필수설명
currentPasswordStringY현재 비밀번호 (RSA 암호화)
newPasswordStringY새 비밀번호 (RSA 암호화)
confirmPasswordStringY새 비밀번호 확인 (RSA 암호화)
RSA 암호화: 모든 비밀번호는 RSA 공개키로 암호화하여 전송해야 합니다. 공개키는 /{clientType}/{version}/auth/getPublicKey API에서 조회합니다.

Response

{
    "code": 1,
    "message": "OK",
    "data": null
}

TypeScript Interface

TypeScript Interface 
/** 비밀번호 변경 요청 */
interface ChangePasswordRequest {
  /** 현재 비밀번호 (RSA 암호화) */
  currentPassword: string;
  /** 새 비밀번호 (RSA 암호화) */
  newPassword: string;
  /** 새 비밀번호 확인 (RSA 암호화) */
  confirmPassword: string;
}

Error Cases

BizException
코드ResponseCode메시지발생 상황
4008PASSWORD_DECRYPT_FAIL비밀번호 복호화에 실패했습니다.RSA 복호화 실패
8001CURRENT_PASSWORD_INVALID현재 비밀번호가 올바르지 않습니다.현재 비밀번호 불일치
8003PASSWORD_CONFIRM_NOT_MATCH비밀번호가 일치하지 않습니다.새 비밀번호와 확인 불일치
8004PASSWORD_SAME_AS_CURRENT새 비밀번호는 현재 비밀번호와 달라야 합니다.동일 비밀번호 사용

보안 안내

비밀번호 변경 완료 후 기존 세션이 무효화되므로 다시 로그인해야 합니다.
