MobileAuthController API 문서

소스 경로: biz/mobileAuth/controller/MobileAuthController.java

참고: 모든 API는 Gateway를 통해 접근합니다. {clientType}은 app 또는 web으로 대체합니다.

목차

  • 1. SMS 인증번호 발송 (insSmsAuth) - 회원가입용
  • 2. 아이디 찾기 SMS 발송 (insFindIdSmsAuth) NEW
  • 3. 비밀번호 찾기 SMS 발송 (insFindPwSmsAuth) NEW
  • 4. SMS 인증번호 검증 (updSmsAuthVerify)
  • 5. SMS 인증번호 발송 - 인증 필수 (insAuthSmsAuth)
  • 6. SMS 인증번호 검증 - 인증 필수 (updAuthSmsAuthVerify)
  • 7. 전체 TypeScript Interface

아이디 찾기 프로세스 가이드

  1. insFindIdSmsAuth API를 호출하여 이름+휴대전화번호 확인 후 인증번호를 발송합니다
  2. 사용자가 수신한 인증번호를 입력합니다
  3. updSmsAuthVerify API를 호출하여 인증번호를 검증합니다 (authType: "FIND_ID")
  4. getFindId API를 호출하여 이메일 목록을 조회합니다

비밀번호 찾기 프로세스 가이드

  1. insFindPwSmsAuth API를 호출하여 이메일+휴대전화번호 확인 후 인증번호를 발송합니다
  2. 사용자가 수신한 인증번호를 입력합니다
  3. updSmsAuthVerify API를 호출하여 인증번호를 검증합니다 (authType: "FIND_PW")
  4. resetPassword API를 호출하여 비밀번호를 재설정합니다
authType 종류:
- 공개 API: SIGNUP(회원가입), FIND_ID(아이디찾기), FIND_PW(비밀번호찾기)
- 인증 필수 API: CHANGE_PHONE(휴대전화변경), CHANGE_PW(비밀번호변경)

1. SMS 인증번호 발송 (insSmsAuth) 공개 API

회원가입 시 사용합니다. 아이디/비밀번호 찾기는 별도 API를 사용하세요.

API URL

POST
/{clientType}/{version}/mobileAuth/insSmsAuth

인증 및 권한

토큰NONE
권한PUBLIC
Header-

Request Body (SendSmsAuthDto)

필드명타입필수검증설명
phoneNumberStringY@NotBlank휴대전화번호
authTypeStringY@NotBlank인증 요청 구분 (SIGNUP)

Response (SendSmsAuthResultDto)

필드명타입필수설명
authKeyStringY인증 키
expireSecondsintY인증번호 만료 시간 (초) - 기본 300초
resendWaitSecondsintY재전송 대기 시간 (초) - 기본 60초

Description

목적

아이디 찾기 시 이름과 휴대전화번호로 계정 존재 여부를 확인한 후 SMS 인증번호를 발송합니다.

특징

  • 계정이 존재하지 않으면 SMS를 발송하지 않고 에러를 반환합니다
  • 재전송은 1분 후에 가능합니다
  • 인증번호 유효시간은 5분입니다
