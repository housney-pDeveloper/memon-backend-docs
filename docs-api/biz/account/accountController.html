아이디 찾기 프로세스 가이드

  1. POST /api/{version}/mobileAuth/insFindIdSmsAuth API를 호출하여 이름+휴대전화번호 확인 후 SMS 인증번호를 발송합니다
  2. POST /api/{version}/mobileAuth/updSmsAuthVerify API를 호출하여 인증번호를 확인합니다 (authType: "FIND_ID")
  3. 인증 완료 후 반환된 authKey를 저장합니다
  4. 아이디 찾기 API를 호출하여 이름과 휴대전화번호로 등록된 이메일(아이디) 목록을 조회합니다

비밀번호 재설정 프로세스 가이드

  1. POST /api/{version}/mobileAuth/insFindPwSmsAuth API를 호출하여 이메일+휴대전화번호 확인 후 SMS 인증번호를 발송합니다
  2. POST /api/{version}/mobileAuth/updSmsAuthVerify API를 호출하여 인증번호를 확인합니다 (authType: "FIND_PW")
  3. 인증 완료 후 반환된 authKey를 저장합니다
  4. 비밀번호 재설정 API를 호출하여 새 비밀번호를 설정합니다

1. 아이디 찾기 (getFindId)

1. API URL

POST
/api/{version}/account/getFindId

URL 예시:
POST /api/v1/account/getFindId

2. API Method

POST

3. 인증 및 권한

토큰NONE
권한PUBLIC
Header-

4. API Parameter (Request Body)

필드명타입필수여부설명 (@Info)
nameString필수이름
phoneNumberString필수휴대전화번호
authKeyString필수인증키 (휴대전화 인증 완료 후 발급된 키)

요청 예시

    "authKey": "SMS-20240101120000000-abcd1234"
}

8. API Description

목적

아이디(이메일) 찾기 API입니다. 이름과 휴대전화번호로 가입된 계정의 이메일(아이디)과 가입일을 조회합니다.

선행 조건

  • 로그인이 필요하지 않음 (공개 API)
  • 휴대전화 인증 완료 필수: insFindIdSmsAuth → updSmsAuthVerify 순으로 호출

2. 비밀번호 재설정 (resetPassword) UPDATED

1. API URL

POST
/api/{version}/account/resetPassword

URL 예시:
POST /api/v1/account/resetPassword

2. API Method

POST

3. 인증 및 권한

토큰NONE
권한PUBLIC
Header-

4. API Parameter (Request Body)

필드명타입필수여부설명 (@Info)
authKeyString필수인증 키 (휴대전화 인증 완료 후 발급된 키)
emailString필수이메일(로그인ID)
newPasswordString필수새 비밀번호 (RSA 암호화)
confirmPasswordString필수새 비밀번호 확인 (RSA 암호화)

요청 예시

{
    "authKey": "SMS-20240101120000000-abcd1234",
    "email": "user@example.com",
    "newPassword": "[RSA 암호화된 새 비밀번호]",
    "confirmPassword": "[RSA 암호화된 새 비밀번호 확인]"
}

보안 안내

  • newPassword와 confirmPassword는 반드시 RSA 공개키로 암호화하여 전송해야 합니다.
  • RSA 공개키는 POST /api/{version}/auth/getPublicKey API를 통해 획득합니다.
  • 비밀번호 정책: 8~20자, 영문/숫자/특수문자(@$!%*#?&) 각 1자 이상 포함

5. API Response

응답 예시 (성공)

{
    "code": 1,
    "message": "OK",
    "data": null
}

8. API Description

목적

비밀번호를 잊어버린 사용자가 휴대전화 인증 후 새 비밀번호로 재설정하는 API입니다.

선행 조건

  • 로그인이 필요하지 않음 (공개 API)
  • 휴대전화 인증 완료 필수: insFindPwSmsAuth → updSmsAuthVerify 순으로 호출
  • RSA 공개키로 비밀번호를 암호화하여 전송해야 함

비밀번호 정책

  • 8자 ~ 20자 이내
  • 영문(대소문자) 1자 이상 포함
  • 숫자 1자 이상 포함
  • 특수문자(@$!%*#?&) 1자 이상 포함
  • 기존 비밀번호와 동일 불가
