TermsController API 문서

소스 경로: biz/terms/controller/TermsController.java

참고: 모든 API는 Gateway를 통해 접근합니다. {clientType}은 클라이언트 유형에 따라 app 또는 web으로 대체합니다.

약관 관리 시스템 개요

  • 약관 버전 관리: 각 약관은 버전이 관리되며, latest_yn='Y'인 최신 버전만 표시됩니다
  • 화면별 약관: SIGNUP, LOGIN 등 화면별로 표시할 약관 목록을 관리합니다
  • 약관 전문 여부: contentYn='Y'는 전문 있음, contentYn='N'은 제목만 (예: 만 14세 이상)
  • 로그인 시 재동의: 필수 약관 버전이 변경되면 로그인 시 termsReagreeRequired=true로 반환됩니다

POST

URL 예시: POST /app/v1/terms/getTermsList

/** 화면별 약관 목록 요청 */
interface TermsListRequest {
  /** 화면 코드 (SIGNUP, LOGIN) */
  screenCode: string;
}

/** 약관 항목 */
interface TermsItem {
  /** 약관 번호 */
  termsNo: number;
  /** 약관 코드 */
  termsCode: string;
  /** 약관 명 */
  termsName: string;
  /** 약관 유형 */
  termsType: string;
  /** 약관 버전 */
  version: string;
  /** 필수 여부 */
  requiredYn: string;
  /** 약관 전문 여부 (Y: 전문 있음, N: 제목만) */
  contentYn: string;
  /** 동의 여부 (로그인 사용자만) */
  agreeYn?: string;
  /** 동의 일시 (로그인 사용자만) */
  agreeDate?: string;
}

/** 화면별 약관 목록 응답 */
interface TermsListResponse {
  /** 화면 코드 */
  screenCode: string;
  /** 약관 목록 */
  termsList: TermsItem[];
}

Description

목적

화면별 최신 약관 목록을 조회합니다.

  • 비로그인 사용자: 약관 목록만 반환
  • 로그인 사용자: 동의 여부(agreeYn, agreeDate) 포함 반환

contentYn 설명

  • contentYn='Y': 약관 전문이 있음 (서비스 이용 약관, 개인정보 수집 등)
  • contentYn='N': 제목만 있음 (만 14세 이상 확인)

/** 재동의 필요 약관 항목 */
interface TermsReagreeItem {
  /** 약관 번호 */
  termsNo: number;
  /** 약관 코드 */
  termsCode: string;
  /** 약관 명 */
  termsName: string;
  /** 약관 버전 */
  version: string;
  /** 필수 여부 */
  requiredYn: string;
  /** 약관 전문 여부 */
  contentYn: string;
}

/** 재동의 체크 응답 */
interface TermsReagreeCheckResponse {
  /** 재동의 필요 여부 */
  reagreeRequired: boolean;
  /** 재동의 필요 약관 목록 */
  reagreeTermsList: TermsReagreeItem[] | null;
}

Description

목적

필수 약관 버전이 변경되어 재동의가 필요한지 확인합니다.

로직

  • 필수(requiredYn='Y') + 최신(latestYn='Y') 약관 조회
  • 사용자가 동의하지 않은 약관 필터링
  • 미동의 약관이 있으면 reagreeRequired=true

사용 시점

  • 로그인 시 응답의 termsReagreeRequired로도 확인 가능
  • 별도 API 호출로 재확인 시 사용
