SignupController API 문서

참고: 모든 API는 Gateway를 통해 접근합니다. 회원가입 API는 공개 API로 JWT 인증이 필요하지 않습니다.

SMS 인증 API 안내

SMS 인증 관련 API는 MobileAuthController에서 처리됩니다.

  • 인증번호 발송: POST /api/{version}/mobileAuth/insSmsAuth (authType=SIGNUP)
  • 인증번호 검증: POST /api/{version}/mobileAuth/updSmsAuthVerify (authType=SIGNUP)

목차

  • 1. 고객사 회원가입 (insClient)
  • 2. 전체 TypeScript Interface

1. 고객사 회원가입 (insClient)

1. API URL

POST
/api/{version}/signup/insClient

URL 예시:
POST /api/v1/signup/insClient

2. API Method

POST

3. 인증 및 권한

토큰 NONE
권한 PUBLIC
Header -

4. API Parameter (Request Body)

1) 모바일 인증 정보
필드명 타입 필수여부 설명 (@Info)
authKey String 필수 모바일 인증 키 (MobileAuthController에서 인증 완료된 키)
2) 고객사 정보 (ClientVO 상속)
필드명 타입 필수여부 설명 (@Info)
bizRegNo String 필수 사업자등록번호 (숫자만, 10자리)
clientName String 필수 고객사명 (200자 이내)
clientStartDate String 필수 회사 개업일 (YYYYMMDD, 8자리)
representativeTelephoneNumber String 필수 대표 전화번호 (숫자만, 9~11자리)
zipCode String 필수 우편번호 (숫자만, 5~6자리)
address String 필수 주소 (300자 이내)
addressDetail String 선택 상세주소 (300자 이내)
3) 사용자 정보
필드명 타입 필수여부 설명 (@Info)
userName String 필수 사용자 이름
email String 필수 이메일
mobileNo String 필수 휴대전화번호 (숫자만, 10~11자리)
gender String 필수 성별 (M/F)
encryptPassword String 필수 암호화된 비밀번호 (RSA 공개키로 암호화된 Base64 문자열)
4) 약관 동의 정보
필드명 타입 필수여부 설명 (@Info)
termsNoList List<Long> 필수 동의 약관 번호 목록

요청 예시

{
    "authKey": "SMS-20260114123456789-a1b2c3d4",
    "bizRegNo": "1234567890",
    "clientName": "예시회사",
    "clientStartDate": "20200101",
    "representativeTelephoneNumber": "02-1234-5678",
    "zipCode": "12345",
    "address": "서울시 강남구 테헤란로 123",
    "addressDetail": "10층 1001호",
    "userName": "홍길동",
    "email": "hong@example.com",
    "mobileNo": "010-1234-5678",
    "gender": "M",
    "encryptPassword": "Base64EncodedRSAEncryptedPassword...",
    "termsNoList": [1, 2, 3]
}

5. API Response

필드명 타입 필수 설명 (@Info)
clientNo Long 필수 고객사 번호
clientCode String 필수 고객사 코드 (E + 7자리 숫자)
userNo Long 필수 사용자 번호
userCode String 필수 사용자 코드 (S + 7자리 숫자)
userClientNo Long 필수 사용자-고객사 관계번호

응답 예시 (성공)

{
    "code": 1,
    "message": "OK",
    "data": {
        "clientNo": 1001,
        "clientCode": "E0000001",
        "userNo": 5001,
        "userCode": "S0000001",
        "userClientNo": 100
    }
}

4004 EMAIL_INVALID 이메일 형식이 올바르지 않습니다. 이메일 형식이 유효하지 않은 경우

8. API Description

목적

고객사 최초 회원가입 API입니다. 모바일 인증 완료 후 고객사 정보와 해당 고객사의 첫 번째 사용자(관리자)를 함께 등록합니다.

선행 조건

  • 로그인이 필요하지 않음 (공개 API)
  • Gateway에서 JWT 인증을 요구하지 않음
  • 모바일 인증 완료 필요: MobileAuthController의 insSmsAuth + updSmsAuthVerify를 통해 인증을 완료하고 발급받은 authKey가 필요합니다
  • 필수 약관 동의 필요: termsNoList에 필수 약관 번호들이 포함되어야 합니다
  • 비밀번호는 RSA 공개키로 암호화하여 전송해야 합니다

기대 효과

  • 새로운 고객사와 해당 고객사의 관리자 사용자 동시 생성
  • 고객사 번호(clientNo), 고객사 코드(clientCode), 사용자 번호(userNo), 사용자 코드(userCode), 관계번호(userClientNo) 발급
  • 회원가입 완료 후 로그인 API를 통해 서비스 이용 가능
회원가입 프로세스:
  1. /auth/getPublicKey API로 RSA 공개키 조회
  2. /mobileAuth/insSmsAuth API로 휴대전화 인증번호 발송 (authType=SIGNUP)
  3. /mobileAuth/updSmsAuthVerify API로 인증번호 확인
  4. /signup/insClient API로 회원가입 완료
