ClientManageController API 문서

인증 필수 (Back-office 전용)

이 Controller의 모든 API는 로그인 상태에서만 호출 가능합니다.

요청 시 Authorization: Bearer {JWT토큰} 헤더가 필수입니다.

권한 정책

  • OWNER: 모든 고객사 조회/수정 가능
  • ADMIN: 소속 고객사만 조회/수정 가능
  • 그 외 권한: 접근 불가
참고: 모든 API는 Gateway를 통해 접근합니다. {clientType}은 클라이언트 유형에 따라 app 또는 web으로 대체합니다.

1. 고객사 목록 조회 (getClientManageList)

1. API URL

POST
/{clientType}/{version}/client/manage/getClientManageList

URL 예시:
POST /app/v1/client/manage/getClientManageList
POST /web/v1/client/manage/getClientManageList

2. API Method

POST

3. 인증 및 권한

토큰 ACCESS_TOKEN
권한 OWNER, ADMIN
Header Authorization: Bearer {accessToken}

4. API Parameter (Request Body)

필드명 타입 필수여부 설명 (@Info)
searchKeyword String 선택 검색어 (고객사명, 사업자등록번호)
useYn String 선택 사용 여부 (Y/N)
pageNum int 선택 페이지 번호 (기본값: 1)
pageSize int 선택 페이지 크기 (기본값: 10)

요청 예시

{
    "searchKeyword": "핀게이트",
    "useYn": "Y",
    "pageNum": 1,
    "pageSize": 10
}

5. API Response

필드명 타입 설명 (@Info)
totalCount int 총 개수
clientList List<ClientVO> 고객사 목록

clientList 항목 (ClientVO)

필드명 타입 설명 (@Info)
clientNo Long 고객사 번호
clientCode String 고객사 코드 (E + 7자리 숫자)
bizRegNo String 사업자등록번호
clientName String 고객사명
clientStartDate String 회사 개업일 (YYYYMMDD)
representativeTelephoneNumber String 대표 전화번호
verifyYn String 인증 여부 (Y/N)
useYn String 사용 여부 (Y/N)
regDate String 등록일시
modDate String 수정일시

응답 예시 (성공)

{
    "code": 1,
    "message": "OK",
    "data": {
        "totalCount": 25,
        "clientList": [
            {
                "clientNo": 1,
                "clientCode": "E0000001",
                "bizRegNo": "123-45-67890",
                "clientName": "(주)핀게이트",

8. API Description

목적

Back-office에서 특정 고객사의 상세 정보를 조회합니다.

선행 조건

  • 로그인 필수: JWT 토큰이 필요합니다
  • 권한 필요: OWNER 또는 ADMIN 권한

권한별 동작

  • OWNER: 모든 고객사 상세 조회 가능
  • ADMIN: 소속 고객사만 조회 가능 (다른 고객사 조회 시 오류)

9. API Exception Case

Case 발생 상황 ResponseCode 메시지
1 존재하지 않는 고객사 번호로 조회 요청 CLIENT_NOT_FOUND (31501) 고객사 정보를 찾을 수 없습니다.
2 ADMIN이 다른 고객사 정보 조회 시도 CLIENT_ACCESS_DENIED (31502) 해당 고객사에 접근 권한이 없습니다.
3 필수 필드(clientNo) 누락 INVALID_PARAMETER (90002) 고객사 번호는 필수입니다.

3. 고객사 정보 수정 (updClientManage)

1. API URL

POST
/{clientType}/{version}/client/manage/updClientManage

URL 예시:

POST /web/v1/client/manage/updClientManage

2. API Method

POST

3. 인증 및 권한

토큰 ACCESS_TOKEN
권한 OWNER, ADMIN
Header Authorization: Bearer {accessToken}

4. API Parameter (Request Body)

필드명 타입 필수여부 유효성 검증 설명 (@Info)
clientNo Long 필수 @NotNull (OnUpdate) 고객사 번호
clientName String 필수 @NotBlank, @Size(max=200) 고객사명
clientStartDate String 필수 @NotBlank, @Size(8), @Pattern(YYYYMMDD) 회사 개업일 (YYYYMMDD)
representativeTelephoneNumber String 필수 @NotBlank, @Size(max=20) 대표 전화번호
zipCode String 선택 @Size(max=10) 우편번호
address String 필수 @NotBlank, @Size(max=300) 주소
addressDetail String 선택 @Size(max=300) 상세주소
useYn String 필수 @NotBlank, @Pattern(Y/N) 사용 여부
changeReason String 선택 @Size(max=500) 변경 사유
수정 불가 항목: clientNo(식별용으로만 사용), bizRegNo, clientCode, 서비스 신청일, 고객사 대표 사용자 정보는 수정할 수 없습니다.

8. API Description

목적

Back-office에서 고객사 정보를 수정합니다. 수정 전 데이터는 변경 이력으로 저장됩니다.

선행 조건

  • 로그인 필수: JWT 토큰이 필요합니다
  • 권한 필요: OWNER 또는 ADMIN 권한

권한별 동작

  • OWNER: 모든 고객사 정보 수정 가능
  • ADMIN: 소속 고객사만 수정 가능 (다른 고객사 수정 시 오류)

기대 효과

  • 고객사 정보가 업데이트됩니다
  • 수정 전 데이터가 tb_client_change_history 테이블에 이력으로 저장됩니다
  • 수정자 정보(modUserNo)와 수정일시(modDate)가 자동 기록됩니다

9. API Exception Case

Case 발생 상황 ResponseCode 메시지
1 존재하지 않는 고객사 번호로 수정 요청 CLIENT_NOT_FOUND (31501) 고객사 정보를 찾을 수 없습니다.
2 ADMIN이 다른 고객사 정보 수정 시도 CLIENT_MODIFY_DENIED (31503) 해당 고객사를 수정할 권한이 없습니다.
3 필수 필드 누락 INVALID_PARAMETER (90002) 필수 입력값이 누락되었습니다.
4 회사 개업일 형식 오류 INVALID_PARAMETER (90002) 회사 개업일은 숫자 8자리(YYYYMMDD)로 입력해주세요.
5 사용여부 값 오류 INVALID_PARAMETER (90002) 사용 여부는 Y 또는 N이어야 합니다.
