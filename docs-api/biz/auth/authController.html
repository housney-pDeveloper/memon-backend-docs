AuthController API 문서

소스 경로: biz/auth/controller/AuthController.java

참고: 모든 API는 Gateway를 통해 접근합니다. {clientType}은 클라이언트 유형에 따라 app 또는 web으로 대체합니다. 인증 관련 API는 공개 API로 JWT 인증이 필요하지 않습니다.

보안 주의사항

  • 이 API들은 인증 없이 접근 가능합니다 (Gateway에서 public-paths로 설정됨)
  • 공개키만 제공하며, 개인키는 절대 노출되지 않습니다
  • 공개키를 LocalStorage에 장기 저장하지 마세요
  • 세션당 또는 로그인 시도당 새로 조회하는 것을 권장합니다

1. RSA 공개키 조회 (getPublicKey)

API URL

POST
/{clientType}/{version}/auth/getPublicKey

URL 예시: POST /app/v1/auth/getPublicKey, POST /web/v1/auth/getPublicKey

인증 및 권한

토큰NONE
권한PUBLIC
Header-

Request Body

요청 파라미터가 없습니다. (빈 body 또는 {}로 호출)

Response (PublicKeyDto)

필드명타입필수설명
publicKeyStringY RSA 공개키 (Base64 인코딩)
algorithmStringY 키 알고리즘 (RSA-OAEP)
paddingStringY 패딩 방식 (OAEPWithSHA-256AndMGF1Padding)
keySizeintY 키 크기 (비트)

응답 예시

{
    "code": 1,
    "message": "OK",
    "data": {
        "publicKey": "MIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8AMIIBCgKCAQEA...",
        "algorithm": "RSA-OAEP",
        "padding": "OAEPWithSHA-256AndMGF1Padding",
        "keySize": 2048
    }
}

사용 가이드

  1. 로그인 페이지 진입 시 이 API를 호출하여 RSA 공개키를 조회합니다
  2. 조회된 공개키로 비밀번호를 RSA 암호화합니다 (JSEncrypt 또는 WebCrypto API 사용)
  3. 암호화된 비밀번호(Base64)를 로그인 API의 encryptedPassword 필드에 설정하여 전송합니다

Description

목적

RSA 공개키 조회 API입니다. 프론트엔드에서 로그인 시 비밀번호 암호화에 사용할 공개키를 반환합니다.

선행 조건

  • 로그인이 필요하지 않음 (공개 API)
  • Gateway에서 JWT 인증을 요구하지 않음

기대 효과

  • 클라이언트에서 비밀번호를 안전하게 암호화하여 전송 가능
  • 네트워크 전송 중 비밀번호 노출 방지
  • HTTPS와 함께 이중 보안 제공
